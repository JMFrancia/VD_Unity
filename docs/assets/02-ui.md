# Assets — UI (2D)

**Spec:** `docs/VoidDay-Spec-unity.md` · **Style guide:** `docs/StyleGuide.md`
**Count:** ~29 sprites (18 icons + 8 chrome + 3 rarity frames)

> **This doc enumerates UI *art assets* (icons, chrome sprites, frames).** The *screen layouts* (HUD arrangement, popup contents, menu structure — spec §12.1–12.4) belong to **`ui_inventory`**, which hasn't run yet. When it does, cross-check surfaces against it (per the asset_list skill). Ids here are the naming authority; ui_inventory references them.

## Format

- **Reference resolution:** 1080 × 1920 portrait (StyleGuide Format). Safe-area margins on HUD for 9:19.5–9:21.
- **Style:** filled, rounded, single-weight icons; chunky rounded chrome, corner radius **24–32px @1080w**; soft drop shadows; touch target **≥120px @1080w** (StyleGuide Type & UI).
- **Icon convention:** **128×128** transparent PNG (proposed); resource icons simple rounded flat symbols.
- **Chrome:** **9-slice** sprites (panel, button, bars, card, toast) so one asset scales to every surface.
- **The void accent** (`#8B5CF6`) appears in UI **only** where UI touches void content — pet menu, egg/hatch, hearts, void-event toasts (StyleGuide Type & UI). Farm UI stays warm/neutral.
- **HUD is screen-space UGUI** (spec §12.1); world-space feedback sprites (progress bar, ready icon, heart) render in world space and **billboard** (spec §12.6).
